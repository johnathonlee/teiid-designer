Eclipse Resources

(from Eclipse documentation)

Resources is a collective term for the projects, folders, and files that exist in the Workbench.

Navigation views provide a hierarchical view of resources and allows you to open them for editing. Other tools may display and handle these resources differently.

There are three basic types of resources that exist in the Workbench:

Files

  • Comparable to files as you see them in the file system.

Folders

  • Comparable to directories on a file system. In the Workbench, folders are contained in projects or other folders. Folders can contain files and other folders.

Projects

  • Contain folders and files. Projects are used for builds, version management, sharing, and resource organization. Like folders, projects map to directories in the file system. (When you create a project, you specify a location for it in the file system.)
  • A project is either open or closed. When a project is closed, it cannot be changed in the Workbench. The resources of a closed project will not appear in the Workbench, but the resources still reside on the local file system.
  • Closed projects require less memory. Since they are not examined during builds, closing a project can improve build time.
  • When a project is open, the structure of the project can be changed and you will see the contents.

Designer implements these concepts via the Metadata Model file and the Model Project, specific to Designer. Models are *.xmi files specifically designed to persist your critical metadata.

Creating a Project

To create a new project in Designer:

Step 1: Select the New action on the main toolbar and choose Model Project or select in the Model Explorer and right-click select the New > Model Project action.

Step 2: Provide a valid project name in the New Model Project dialog.

Step 3: On the final page, choose any initial project dependencies or referenced projects you wish to establish. Click Finish when done.

Add Existing Project(s)

To add an existing project or projects to your workspace:

Step 1: Select the File > Add Existing Project action.

Step 2: Select an existing root directory containing one or more existing projects via the Browse... button or provide a valid folder by typing in the text field.

Step 3: Check one or more existing projects in the Projects tree.

Step 4: Click Finish.

Add Existing Model Project Set

A Model Project Set is a zip file containing one or more valid Model Projects. (see Export Model Project Set)

To add an existing model project set to your workspace:

Step 1: Select the File > Add Existing Model Project Set action.

Step 2: Browse... your file system and select and existing Model Project Set zip file.

Step 3: Specify a target folder to copy this project into. (usually your installation's workspace folder). Type in the location or select Browse.. to select a folder on your local file system.

Step 4: Change/set any desired options

Step 5: Select Finish.

Export Model Project Set

To export an existing model project set from your workspace:

Step 1: Select the File > Export... action, or select in the Model Explorer and choose Export... action.

Step 2: In the Export dialog, choose the Model Project Set option.

Step 3: The next page, select which project sets in your workspace you wish to include in the Model Project Set.

Step 4: Select Finish.

Create a Folder

To create a new folder in Designer:

Step 1: Select the New action on the main toolbar and choose Folder or select in the Model Explorer and right-click select the New > Folder action.

Step 2: Provide a valid project or parent folder name in the Folder dialog, or select a folder in the workspace tree.

Step 3: Provide a unique folder name.

Step 4: Click Finish.
