Enterprise Designer Perspectives

Enterprise Designer utilizes Eclipse's Workbench environment which controls visual layout via perspectives. A perspective defines the initial set and layout of views and editors. Within the application window, each perspective shares the same set of editors. Each perspective provides a set of functionality aimed at accomplishing a specific set of tasks. 

Perspectives control what appears in certain menus and toolbars. They define visible action sets, which you can change to customize a perspective. You can save a perspective that you build in this manner, making your own custom perspective that you can open again later. 

Enterprise Designer perspectives include:

  • Designer - Default perspective providing fundamental model editing and management capability.
  • SQL Explorer- Provides views and actions to perform tests on your VDB by connecting to your data sources and executing your test queries.
  • Metabase Repository - Provides views and actions related to storing, retrieving and maintaining models, VDBs and projects in the Metabase Repository.

Each perspective utilizes Designer's Main Menu/Toolbar.

Designer Perspective

The Designer perspective includes 4 main UI components or groups of components as shown below. They include:

  • Model Explorer
  • Properties View
  • Model Editors
  • Miscellaneous Views

 

Metabase Repository Perspective

The Metabase Repository perspective includes 3 main UI components or groups of components. They include:

  • Repository Explorer
  • Repository Contents Editor
  • Properties View
  • Miscellaneous Views

SQL Explorer Perspective

The SQL Explorer perspective includes 5 main UI components or groups of components. They include:

  • Database Structure View - provides an updated view of your modeled data as seen from the perspective of your VDB.
  • SQL History View - As you execute queries against your data, a history of executed SQL text is maintained in this view.
  • Connections View - You can see any current connections to your VDBs.
  • SQL Editor - Specify SQL and submit queries from this editor.
  • SQL Results View - shows your query results in table-row format for relational models and XML format for Web Service or XML document queries.

Note: The SQL Explorer perspective will be opened whenever you Test your VDB.

Opening a New Perspective

There are a couple of ways to open a new perspective:

  • Using the Open Perspective button on the shortcut bar.
  • Choosing a perspective from the Window > Open Perspective menu. 

To open a perspective by using the shortcut bar button:

Step 1: Click on the Open Perspective button . 

Step 2: A menu appears showing the same choices as shown on the Window > Open Perspective menu. Choose Other from the menu.

Step 3: In the Select Perspective dialog choose Designer and click OK.

The Designer perspective is displayed.  

Step 4: There are several other interesting things to take note of.

    The title of the window now indicates that the Designer perspective is in use.

The shortcut bar may contain multiple perspectives. The perspective button which is pressed in, indicates that it is the current perspective.

To display the full name of the perspectives, right click the perspective bar and select Show Text.

To change perspective, select the desired perspective button. Notice that the set of views is different for each of the perspectives.
