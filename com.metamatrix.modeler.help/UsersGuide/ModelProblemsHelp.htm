Model Validation

Validation Nuts and Bolts

Validation is the process performed in Designer which verirfies that models and VDBs within your workspace adhere to pre-determined specifications. These specifications are typically are model-specific and cover everything from grammar in SQL and object naming constraints, to various properties. Some of these validation specifications can be customed via User Preferences.

There are a number of ways that resources in the workspace can be validated. The scope of a validation or "Build", can be one or more selected projects, a working set, or the entire workspace.

A build will typically only operate on resources that have changed since the last build. A clean build will discard all existing built state, causing the next build to operate on all resources within the scope of the build.

Builds can be done automatically (each time resources are modified), or manually, using a menu item or keyboard shortcut. See the Related tasks links for more details.

Assessing your validation results

Validation errors or warnings may occur during a modeling session in Designer due to the transitional nature of editing models. Creating new objects that require at least one child, for example, will usually create a validation error. If Models > Build Automatically is turned ON, then validation will occur with each Save operation, otherwise validation has to be performed manually via Project > Build All, Project >Build Project, Project > Build Working Set or Project > Clean... actions.

Note: Validation errors may prevent you from executing your VDB.

These problems are summarized in the Problems View and may be one of two types:

Error
Warning

You can navigate to and view the specific problem area by double-clicking the error or warning in the Problems View. In most instances, the Model containing the error will open in an editor and the specific problem object will be visible and selected.

Users can also navigate to problem area by selecting row and right-click select "Go To" action.

Enterprise Designer provides a number of Validation preferences you can change to suit your particular database environment.

Clean.... Build

To perform a clean build:

Step 1: Select Project > Build Working Set or Project > Clean... action from the main menu.

Step 2: Select either Clean all projects or ....

Clean all projects selected below option and check/uncheck projects you wish to build or omit from re-building.
