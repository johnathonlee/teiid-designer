The MetaMatrix Repository

The MetaMatrix Repository is a graphical metadata management system in Designer that helps you store, import, organize, and report on the diverse metadata that supports your data service effort. You can assemble all the metadata that supports your data service work, and share metadata, models, and data service definitions across teams and projects for a clearer picture of how enterprise data is interrelated.

This repository can be viewed in Designer via the MetaMatrix Repository perspective. You can open this perspective any time, via the shortcut toolbar button. ( See Opening a New Perspective)

To use the Repository, you'll need to create repository connections through which you'll share and control your projects and resoures. You can do this in two ways:

  • Open the Repository perpective and create repository connections prior to sharing/adding resources.
  • Using the Team > Share Project action - provides step-by-step creation of repository connections and adds resources to repository.

Create New Repository Connection

To create a repository connection:

Step 1: Open the Repository perspective via the shortcut toolbar button

Step 2: Right-click in the Repository Explorer tree and select New MetaMatrix Repository action. You can also select this action from the main toolbar's New button

Step 3: Enter a name and critical authentication details for accessing the repository and click Finish > to establish the connection.

Note: You can check user authentication by selecting the Test Connection button.

Step 4: Enter a valid pasword and click OK.

Sharing Projects and Resources

To share either a model or an entire project to the team repository, follow the steps below:

Step 1: Right-click on a model or project in the Model Explorerselect Team > Share Project… from the pop-up menu.

Step 2: The Share Project dialog box displays. If you have access to existing repositories, they will be selectable through the dialog box. For the purposes of this example, we will be creating a new repository. Select the radio button for Share in a new repository and click NEXT >.

NOTE: The following procedures assume that you are using the MetaMatrix MetaMatrix Repository.  If you are using another third-party repository the following dialog boxes do not apply.

Step 3: Enter a name and critical authentication details for accessing the repository and click Finish > to establish the connection.

Note: You can check user authentication by selecting the Test Connection button.

Step 4: Enter a valid pasword and click OK.

Step 5: Next, a dialog is presented which provides options to add project resources to the repository at this time. You can supply an initial comments and select from a set addtional options to add resources related by structure (subfolders) or dependency (imported models). In addition, you have an option of deleting your local copies of your resources after they are added to the repository.

Click OK, when finished, to add resources, or Cancel if you do not wish to add resources at this time.

  • Recurse Subfolder – checking this option imports everything at a entity level and all subsequent sublevels. If you are adding a model, this option will be grayed out.
  • Include Imported Models – if any projects are linked to model imports, selecting this option will capture the necessary imported models as well.
  • Remove local copy – checking this option deletes your local project or model file after it is added to the database, ensuring the existence of only one version at a time.
