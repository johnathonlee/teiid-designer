New Model Builders

The New Model Wizard provides optional ways to build new models from existing models or other file system or URL resources. This section summarizes these capabilities.

Each type of model supported by Designer provides one or more model builder options as listed below.

  • Relational Source
    • Copy from existing model
    • Generate from existing UML models
  • Relational View
    • Copy from existing model
    • Transform from existing model
    • Generate from existing UML models
  • XML Service Source
    • Copy from existing model
  • XML Service View
    • Copy from existing model
  • XML Document View
    • Copy from existing model
    • Build XML documents from XML schema
  • XML Schema
    • Copy from existing model
  • Web Service
    • Copy from existing model
    • Build from existing WSDL file(s) or URL
  • Model Extension
    • Copy from existing model
  • Relationship
    • Copy from existing model
  • Uml 2.0
    • Copy from existing model
  • People
    • Copy from existing model

 

Copy from existing model

This builder option performs a structural copy of the contents of an existing model to a newly defined model.

When this builder is selected and the New Model wizard's Next > button is clicked, the Copy an Existing Model page is displayed.

Step 1:  Select an existing Relational model from the workspace using the browse button .

Note: An existing model will be pre-selected if a Relational model in the workspace is selected in the VDB explorer prior to starting the new model wizard.

Step 2: If an exact copy of the existing model is wanted, ensure Copy entire model is selected. If all model entities should not be copied, select the Select Model Features to Copy option and use the tree to make your selections.

 

Transform from existing model

This builder is only applicable for creating a relational view model from a relational source model with the added feature of creating default transformations (SELECT * FROM SourceModel.Table_X) for each source table. The steps are the same as Copy Model From Existing Model described above.

Build XML documents from XML schema

This builder option creates an XML View document model based on a selected XML schema and it's dependencies.

When this builder is selected and the New Model wizard's Next > button is clicked, the Select XML Schema page is displayed.

Step 1: Select an existing XML schema model from the workspace using the browse button .

Note 1: An existing model will be pre-selected if an XSD model in the workspace is selected in the VDB explorer prior to starting the new model wizard.

Note 2: The schema must be found in the workspace so if you need to get one or more into the workspace use the XSD Schemas on file system importer.

Step 2: Move the available schema root elements you want to become virtual documents in the new model over to the Virtual Documents list by using the arrow button for selected elements or the button to move all elements.

Step 3: Select the appropriate document options and mapping options.

Step 4: Click Finish to create a model of all selected document entities or click Next > to view Selected Documents Statistics page which shows document entity statistics and gives you an idea the size of the model being created.

Step 5: (Optional) Click Finish to create a model of all selected document entities or click Next to view Preview Generated Documents page that allows you to exclude document entities then click Finish.

 

Build from existing WSDL file(s) or URL

This builder option creates a Web service model based on a user-defined WSDL file and it's referenced schemas. In addition, applicable XML schema files are created and an XML View document model (optional).

Remaining wizard steps are identical to those found using the Import > WSDL File into Web Service Model action option.
