Components

Enterprise Designer allows you to store, organize, and report on your diverse metadata via the MetaMatrix Repository. The Repository Explorer located in the Repository Perspective, provides a simple file-structured view of your respository resources.

The Repository Explorer (shown below) is comprised of a toolbar and a tree view and works in conjunction with the Repository Contents table, Repository Console and Repository Status views.

The Repository Contents view presents the contents of the selected resource in the Repository Explorer. This view provides structural navigation via the up folder icon on it's toolbar and the use of a mouse double-click action. The Location field indicates the current container.

As Repository actions are performed (Share, Check-In/Check-Out), the Repository Console provides feedback on the status/success of the action.

The Repository Status view summarizes checked-out status for all models associated with all connected repositories.

Selection-Based Action Menus

Selecting specific objects in the Repository Explorer provides a context from which the Designer presents a customized menu of available actions. Note that these identical actions are available in the Repository Contents table.

Selecting a repository connection and righ-clicking reveals the following menu:

Selecting a folder or resource (i.e. model) enables a number of team-based actions including:

  • Get Latest...- retrieves the latest version selected projects or models in the repository.
  • Check In... - adds a copy of a project or resource on your local file system (your workspace) to the shared repository.
  • Check Out... - retrieves the latest version selected projects or models in the repository and locks the resource(s) in the repository.
  • Undo Checkout... - unlocks the selected projects or models in the repository so that others can access and modify the current versions.
  • Remove from Repository... - deletes the selected resource(s) and their children from the repository.
  • Show Differences... - provides difference analysis between projects and models in the repository and your local copies of your resources.
  • Show History... - provides a history of a selected resource in the repository, including comments, check-out status and users.
  • Refresh Checkout Status... - refreshes the status of a resource in the Repository Explorer. This is required if multiple users are working in the same repository.
  • Refresh Folder - refreshes the content of your Repository Explorer. This is required if multiple users are working in the same repository
  • New MetaMatrix Repository - create new repository connections.

Get Latest...

The following dialog provides the get latest version options available for your resource(s). Set your options and click OK to get copies of your resources.

Options within this dialog box include:

  • Recurse Subfolder – checking this option return everything at a entity level and all subsequent sublevels.
  • Include Imported Models – if any projects are associated with model imports, selecting this option will enable the necessary imported models as well.
  • Make Writable – enables the contents of the project model to be modified within the workspace.
  • Replace Writable: – determines what will happen when a writable model already exists in the workplace.
    • Ask – prompts you whether or not to replace existing writable models in the workspace.
    • Replace – replaces existing writable models in the workspace.
    • Skip – keeps all modifications temporary, and does not actively save them unless you select File > Save in the Designer.

Check In...

The following dialog provides the check-in options available for your resource(s). Set your options and click OK to check in resources.

Options within this dialog box include:

  • Recurse Subfolders – check in the entire contents of the project or folder, including the contents of child subfolders.
  • Include Imported Models – check in any models imported by the models already selected to be checked out.
  • Keep checked out – if you want to place a version of this file in the Repository, but you want to keep it locked and checked out for more editing.
  • Remove local copy – deletes the checked in files from your workspace.
  • Use check out comment – enables you to associate a notation with this action.

Check Out...

The following dialog provides the check-out options available for your resource(s). Set your options and click OK to check out resources.

Options available within this dialog box include:

  • Recurse Subfolder – Undo check out for the entire contents of the project or folder, including the contents of child subfolders.
  • Include Imported Models – Undo check out for any models imported by the models already selected for undo check out.
  • Don’t get local copy – checking this option will mark the selected files as checked out without copying those files to your workspace.
  • Replace Writable– Determines what happens to the copies in your workspace of the files to be undone.
    • Replace – Overwrites the copy in your workspace with the one from the Team Share Repository.
    • Leave – Releases the lock on the file, but leaves the local copy (including any changes you made) intact.
    • Delete – Deletes the local copy of the file from your workspace.

When finished, the affected resource (model) will display a check decorator indicated a checked-out status in the Repository Explorer, Repository Contents view and the Model Explorer (see below).

 

Undo Checkout...

The following dialog provides the undo check-out options available for your resource(s). Set your options and click OK to undo check out resources.

Options available within this dialog box include:

  • Recurse Subfolder – Undo check out for the entire contents of the project or folder, including the contents of child subfolders.
  • Include Imported Models – Undo check out for any models imported by the models already selected for undo check out.
  • Local Copy – Determines what happens to the copies in your workspace of the files to be undone.
    • Replace – Overwrites the copy in your workspace with the one from the Team Share Repository.
    • Leave – Releases the lock on the file, but leaves the local copy (including any changes you made) intact.
    • Delete – Deletes the local copy of the file from your workspace.

Remove from Repository...

This action deletes the selected resource(s) from the repository. A confirmation dialog is provided to insure delete was intended.

Show Differences...

The Show Differences action performs a comparison of the selected resource in the repository and the corresponding local version of that resource. The following dialog displays the difference results and indicates what has been changed, added and/or removed. Selecting an object (i.e. difference) will display details of the change in the Selection Details section.
