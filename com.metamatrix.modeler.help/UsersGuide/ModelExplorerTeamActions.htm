Team Actions

Enterprise Designer allows you to perform MetaMatrix Repository operations from the Model Explorer via the Team action set.

These actions include:

  • Get Latest...- retrieves the latest version selected projects or models in the repository.
  • Check In... - adds a copy of a project or resource on your local file system (your workspace) to the shared repository.
  • Check Out... - retrieves the latest version selected projects or models in the repository and locks the resource(s) in the repository.
  • Undo Checkout... - unlocks the selected projects or models in the repository so that others can access and modify the current versions.
  • Add to Repository... - addes the selected resource(s) and their children (if appropriate options set) to the repository.
  • Remove from Repository... - deletes the selected resource(s) and their children from the repository.
  • Show Differences... - provides difference analysis between projects and models in the repository and your local copies of your resources.
  • Show History... - provides a history of a selected resource in the repository, including comments, check-out status and users.
  • Refresh Checkout Status... - refreshes the status of a resource in the Repository Explorer. This is required if multiple users are working in the same repository.

The following are additional actions available when selecting a Project.

  • Repository properties- presents a read-only dialog summarizing the selected Repository connection properties.
  • Share Project ... - provides difference analysis between projects and models in the repository and your local copies of your resources.
  • Unshare Project - unshares the selected project from it's current shared repository.

The majority of these actions are described in the Repository Explorer section.

Add to Repository

The following dialog provides the Add Files options available for your resource(s). Set your options and click OK to add copies of your selected resources.

Note that this will result in an initial check-in of your resources, so they will be changed to read-only status.

Options within this dialog box include:

  • Recurse Subfolder – checking this option imports everything at a entity level and all subsequent sublevels. If you are adding a model, this option will be grayed out.
  • Include Imported Models – if any projects are linked to model imports, selecting this option will capture the necessary imported models as well.
  • Remove local copy – checking this option deletes your local project or model file after it is added to the database, ensuring the existence of only one version at a time.
