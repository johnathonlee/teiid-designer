Importing ERwin source

Designer provides an option to import ERwin files (versions 3, 3.5.2, 4, 4.1, and 7). The importer will create Relational models in the workspace, as well as Logical, Relationship and Datatype models if applicable.

Supported structural features of ERwin models include:

  • Schema
  • Catalog
  • Table
  • View
  • Column
  • Index
  • Procedure
  • Parameter
  • Foreign key
  • Primary key
  • Unique key
  • Logical relationship

Any properties (name-value pairs) of these features that are unique to your use of ERwin (or describe physical database attributes) will be implemented using the Designer's metadata extension functionality. The Designer will allow the display and editing of the imported metadata in UML notation in package diagrams within the model.

To import an ERwin file, follow the steps below.

Step 1: In Model Explorer , choose the File > Import action in the toolbar or select a project, folder or model in the tree and choose " Import..."

Step 2: Select one of the ERwin import options below, then choose NEXT>

  • Metadata from ER/Studio 5.1 to 7.1 Models
  • Metadata from ERwin 3.0 to 3.5.2 Models
  • Metadata from ERwin 4.0 SP1 to 4.1 Models
  • Metadata from ERwin 7 Models

Step 3: Click on the Browse... button to locate and select your ERwin Source Model and choose NEXT>

Any user-defined properties and values will be at the top of the information listed in the User Properties and Import Options section. A number of core properties and values will always appear.

Step 5: Specify valid "Location" by editing the text field, or select Browse... button to locate and select a folder or project in your workspace.

Step 6: Modify/Specify names of generated models.

Step 7: Click Finish.

Notes:

  • When you import an ERwin model, all entities from the ERwin model that are Logical Only or Logical and Physical get created in the Logical (UML) model.
  • Also, all entities from the ERwin model that are Physical Only or Physical and Logical get created in the Source Model.
  • All types used are created in a Datatype schema.
  • All relationships between entities (Primary Keys, Foreign Keys, Associations, etc) are represented in the Relationship model and may also be represented in another model.
