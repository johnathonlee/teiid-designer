Reconciling Transformations

When you have modified a transformation or the target virtual class, you might create discrepancies between the symbols you enter in the transformation and the attributes within your virtual metadata class. These descrepancies will typically reflect unbound attributes, unmatched SQL symbols and/or mis-matched datatypes. You can manually resolve these discrepancies using the Reconcile Virtual Target Attributes dialog.

 

The table on the left of the dialog contains the a list of all virtual target attributes and their bound SQL symbols. This defines the mapping or "transformation" of your source data into your virtual, or view table.

You can lock your target attributes for virtual models that your enterprise already uses with its applications, especially in situations where renaming an attribute might render the application inoperative.

Reconcile Target Attributes

To reconcile your transformations target attributes

Step 1: Open the transformation diagram.

Step 2: Click the Reconcile Target Attributes button to open the dialog (located on the Diagram Toolbar and the Transformation Editor toolbar)

Step 3: If the unmatched target attributes, datatype conflicts, or unmatched SQL symbols number is not 0, you need to reconcile your transformation.

If your virtual attributes are locked, the Target Locked check box contains a checkmark. You can click this check box to temporarily remove the checkmark so that you can modify the virtual attributes. This checkbox helps to prevent you from inadvertently modifying a virtual attribute, which your production applications might use.

If your virtual attributes are unlocked, you can reorganize the attributes in the target class as well as add or remove attributes based upon the symbols you have typed in your transformation.

Step 4: Use this dialog to match the number of rows in the Target Group Attributes table, which lists the attributes defined in your virtual class, with the number of items in the Unmatched SQL Symbols list box, which lists the symbols in your query that do not reflect existing attributes in your virtual metadata.

As you make changes, the SQL Query Display contains your reconciled query.

You can then:

  • Add a new attribute based on a SQL symbol.You can only do this if your target class is unlocked. In the Unmatched SQL Symbols list box, select the symbol you want for which you want to create an attribute and click New. The item displays in the Target Group Attributes table and the Designer will create an attribute meta object for this symbol.
  • Delete an existing attribute from the virtual class. You can only do this if your target class is unlocked. Click the row in the Target Group Attributes table that you want to delete and click Delete. The Designer removes this row from the table and this attribute from your virtual class.
  • Reorder the attributes in the virtual class. You can only do this if your target class is unlocked. Click a row in the Target Group Attributes table that you want to move and click Up to move the attribute (and its symbol) toward the beginning of the list or Down to move the attribute (and its symbol) toward the end of the list.
  • Bind an unmatched symbol to an existing target attribute. This simply renames the unmatched symbol to match an existing attribute. In the Unmatched SQL Symbols list box, select the symbol to bind. In the Target Group Attributes table, select the attribute to which you want to bind the symbol. Click < Bind. The Designer renames the symbol in your query and displays the symbol, renamed to match the target attribute, in the table.
    • NOTE: You can select a row in the table and click Unbind > to divorce a symbol from its target attribute.
  • Add SQL symbols. This action allows adding any available SQL Element Symbol from your query source tables into your table and the query text.
    • NOTE: In cases where there are different sources but same symbol names, you can highlight the tooltip for these symbols and view the full path name including model (see below).

  • Remove selected unmatched symbols. This removes the selected unmatched symbol from the query text and the Unmatched SQL Symbols table.
  • Clear all unmatched symbols. This removes all remaining unmatched symbols from the query text and the Unmatched SQL Symbols table.
  • Sort/Unsort the Unmatched SQL Symbols table. Sort orders the unmatched symbols alphabetically by name, to make it easier to find a specific symbol. Unsort restores them to their natural order.
  • Resolve the Datatype. If there are any datatype conflicts, you can change the datatype or use a CONVERT function to alter the output of the query. Select a row and click Type Resolver … to review the options.

Step 5: After you have reconciled all unmatched SQL symbols and target attributes, click OK.


The Designer reconciles the query as you specified. This dialog box can alter the Select statement’s output by removing or aliasing the existing SQL symbols. If you want to add or remove attributes to the query, you must use the Transformation Editor.

Type Resolver

Your SQL Bindings may require resolving mismatched datatypes. The figure below provides a simple example of a target attribute with type string bound to a column of type short.

To resolve this mismatch:

Step 1:  Select the Type Resolver.... botton to launch the Resolve Datatype Conflicts dialog.

Step 2 :  Select the Attribute-SQL Binding you wish to edit.

Step 3 :  Choose which aspect of the binding you wish to edit. You can either:

  • Change the datatype of the target attribute (i.e. runtime type) by selecting an appropriate datatype via the Change button, then selecting Apply
  • Change the SQL Symbol to perform a convert() function by selecting Apply. Typically the conversion will be pre-constructed for you.
Step 4: Select OK when finished. Note, as each binding is corrected, it should be reflected by a change in Status and color of the row in the table.
