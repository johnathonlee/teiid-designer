Modeling Your Data

Models can be created in different ways. The two primary methods include:

  • Import Wizard
  • New Model Wizard
  • Other New Model options

Import Wizard

The Import Wizard provides a means to create a model based on the structure of a data source, to convert existing metadata (i.e. WSDL or XML Schema) into a source model or to load existing metadata files into the current VDB.

The following import options are available in Designer:

  • DDL into Relational Model
  • Metadata from IBM Rational Rose (c) Model
  • Metadata from JDBC Database
    • User defines which JDBC data source to retrieve metadata
    • Relational source model is created representing that data source
  • Metadata from Text Files on file system
  • Salesforce as Relational Source Model
  • WSDL into Relational Source Model
  • WSDL into Web Service Model
    • Select WSDL file on file system or provides valid WSDL URL
    • An XML View model is created representing the XML response document
    • Input and Output XML Schema (xsd) models are created defining the format of the response document
    • A Web Service model containing interfaces and operations defined in the WSDL mapped to the XML View documents
  • XSD Schemas
    • Select XML Schema (xsd) file(s) on file system
    • Selected XML Schema files and their dependent schemas are copied and loaded into the current VDB

To launch the Import Wizard, choose the File > Import action or select a project, folder or model in the tree and right-click choose "Import..."

New Model Wizard

The New Model Wizard provides a another valuable means to create models of varying types.

Teiid Designer currently supports creating the following types of models:

  • Relational Source
  • Relational View
  • XML Document View
  • XML Schema
  • Web Service
  • Model Extension
  • Uml 2.0
  • Relationship

Use one of the following options to launch the New Model Wizard.

  • Choose the File > New... > Teiid Metadata Model action
  • Select a project or folder in the Model Explorer and choose the same action in the right-click menu.
  • Select the New button on the main toolbar and select the  Teiid Metadata Model action

 

To create a new model:

Step 1: Provide a unique model name, model class (i.e. either Relational, XML, Web Service etc...) and model type (i.e. Source or View)

Step 2: Click Finish OR select an optional model builder and click Next > to continue. Additional steps will vary depending on the builder selected. See New Model Builders.

 

Other New Model options

There are a few additional options or actions available which create new models.

Create Relational View Model from Schema

Step 1: Select any XML Schema model (XSD) in the Model Explorer tree.

Step 2: Right-click to display the context menu and select the "Modeling > Create Relational View Model from Schema" action.

Step 3: In the "Create Virtual Tables form XSD Schema Wizard" dialog, specify a relational view model name name. Click "Finish" when done.

Step 4: Select one or more global schema entities and move from left panel to the right panel. Click "Finish" when done.

Step 5: When model generation is complete the new model will be opened in an editor for your viewing..
