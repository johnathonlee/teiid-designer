Create XML View Model in Designer

XML View models transform relational source data into XML views. You can create XML View models using the steps below...

Step 1: Launch the New Model Wizard dialog by either choosing the File > New... > Metadata Model action , selecting a project or folder in the Model Explorerand choosing the same action in the right-click menu or selecting the New button on the main toolbar and selecting the Metadata Model action

Step 2: Specify a model name.

Step 3: Set Model Class to “XML”.  When you do this, the type will automatically be set to View Model.

Step 4: [Optional]: If you have a similar model in your workspace, you may select the “Copy from an existing model of the same metamodel” builder, then click NEXT> and specify the model you wish to use on the next page.

Step 5: [Optional]: Or, if you have an XML Schema (XSD) file in your workspace, you may select the “Build XML documents from XML schema” builder, then click NEXT> and specify the schema file you wish to use on the next page.

Note: This option will present an additional wizard which guides you through building XML View model components. You will have various options to create a full or partial set of XML Documents from your selected XML Schema file.

Step 6: Click Finish.  After generation the new model can be found in your VDB’s View folder.
