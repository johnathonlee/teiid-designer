Searching In Designer

Designer provides multiple search actions located via Teiid Designer sub-menu in Eclipses Search menu.

The individual search actions are described below:

  • Find Model Object - Launches the Find Model Object dialog, which can be used to find an object in the workspace by specifying all or part of its name.  Selecting the object will open it in the appropriate editor..
  • Relationship... - Launches the Search dialog which auto-selects the Relationship search tab.  Users can search for relationships in the workspace by specifying an Object Type, and/or a Data Type, and/or a property value.  Search results appear in the Search Result view, and double-clicking a result will open that model in the appropriate editor.
  • Tranformation... - Launches the Search Transformations dialog.  Userd can search for transformations in the workspace which contain a specific string within it's SQL text.  Search results appear within the dialog. Users can also preview theSQL for each search result and open a selected result transformation for editing.
  • Metadata... - Launches the Search dialog.  Users can search for models in the workspace by specifying an Object Type, and/or a Data Type, and/or a property value.  Search results appear in the Search Result view, and double-clicking a result will open that model in the appropriate editor.

Finding Model Objects

To find a model object,

Step 1: Open the FInd Model Object dialog by either selecting the Search > Teiid Designer > Find Model Object action on the main toolbar.



Step 2: Begin typing a word or partial word in the Type Object Name field. Wild-card (*) characters will be honored. As you type, the objects which match the desired name will be displayed in the Matching Model Objects list. If there are more than one objects with the same name, the locations or paths of the objects are displayed in the Locations list.

Step 3: If more than one object exists with the desired name, select the one of the locations.

Step 4: Click OK. If editor is not open for the object's model, an editor will open. The desired object should end up displayed in a diagram (if applicable) and selected.

Searching

To search models in Designer:

Step 1: Select Search > Teiid Designer > Metadata... action which opens the Search dialog.

Step 2: Specify desired search options for Object Type, Data Type and Properties.

Step 3: Click Search. The search will be performed and the results will be displayed in the Search Results View. If the view is not yet open, it will be opened automatically.

In addition to searching models based on properties, you can search for relationships via the Relationships search tab.

 

Searching Transformations

To search for string values in your transformations SQL:

Step 1: Select Search > Metadata... action on the main Designer toolbar which opens the Search Transformations dialog.

Step 2: Specify a string segement in the "Find:" field and specify/change your case sensitive preference.

Step 3: Select Perform Search button. Any transformation object containing SQL text which contains occurances of your string will be displayed in the results section.

You can select individual objects and view the SQL. If a table or view supports updates and there is insert, update or delete SQL present, you can expand the object and select the individual SQL type as shown below.

If you wish to view the selected object and it's SQL in a Model Editor, you can click the "Edit" button.

 

Viewing Search Results

Below is an example set of search results. The view contains rows representing matches for your search parameters. You can double-click a entry and the object will be opened and selected in an editor and/or the VDB Explorer if applicable.

The toolbar actions for the Search Results view are:

  • Show Next Match - navigates down one row in the view.
  • Show Previous Match - navigates up one row in the view.
  • Remove Selected Matches - removes selected results from the view.
  • Remove All Matches - Clears the view.
  • Search - Launches the Designer Search Dialog.
  • Previous Search Results - select previous search results from history.
